# before-after-billing

This example explains what a typical project may look like before and after using the `ship-billing` skill.

## Before

A typical project before using this skill may have:
- no pricing page
- no checkout flow
- no subscription status view
- no webhook handling
- no success or cancel pages
- inconsistent billing-related structure

## After

After using the `ship-billing` skill, the project should usually have:
- a clear pricing page
- a clear checkout flow
- a subscription status view
- webhook handling support
- success and cancel pages
- a more consistent billing workflow structure

## What this example is for

This example helps users quickly understand the practical value of the skill before reading the full skill files.
