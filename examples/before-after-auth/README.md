# before-after-auth

This example explains what a typical project may look like before and after using the `ship-auth` skill.

## Before

A typical project before using this skill may have:
- no sign up page
- no sign in page
- no forgot password flow
- no email verification flow
- no protected page example
- inconsistent auth-related structure

## After

After using the `ship-auth` skill, the project should usually have:
- a clear sign up page
- a clear sign in page
- a forgot password flow
- an email verification flow
- at least one protected page example
- a more consistent auth workflow structure

## What this example is for

This example helps users quickly understand the practical value of the skill before reading the full skill files.
