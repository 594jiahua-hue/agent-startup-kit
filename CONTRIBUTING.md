# Contributing

Thanks for your interest in contributing to agent-startup-kit.

## What to contribute

We welcome improvements in:
- skill clarity
- workflow structure
- adapter documentation
- template guidance
- examples and checklists

## Contribution rules

1. Keep changes simple and easy to review.
2. Prefer reusable workflows over project-specific hacks.
3. Do not hardcode secrets, API keys, or credentials.
4. Keep instructions clear for AI coding tools.
5. Update related README or checklist files when needed.

## Before opening a pull request

- explain what was changed
- explain why the change is useful
- keep the scope focused
- avoid unrelated edits

## Style

- prefer short and clear documentation
- keep naming consistent
- optimize for practical usage
